SVG as <img>

function supportsSVG() {
        return document.implementation.hasFeature( "http://www.w3.org/TR/SVG11/feature#Image", "1.1" );
    }

    if( !supportsSVG() ) {
        $( 'img' ).each( function() {
            var src = $( this ).attr( 'src' );
            if( src.indexOf( '.svg' ) !== -1 ) {
                newSrc = src.replace('.svg', '.png');
                $(this).attr('src', newSrc);
            }
        } );
    }

Embeding SVG with <object>

  This will only render if the browser does not support SVG
</object>

Advantages of <object>

  • Caching
  • Scripting Control
  • Embedded Stylesheet Control

Comparison

  Icon Font SVG
Vector Yes, (anti-aliasing, single color) Fully
CSS Control Some Better
Positioning Good Luck Possible
Failures Yes No (if SVG Supported)
Semantics None Better <symbol>
Accessibility Possible Better, can add more info
Browser Support IE 6 + IE 9 +

Possible <svg> Effects

Line Animation

  • A <path> can have a stroke
  • That stroke can be dashed (and animated)
  • The dash can be any length (including the length of the entire path)
